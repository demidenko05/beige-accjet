Beigesoft™ Enterprise Information System Privacy Policy.

All Beigesoft™ EIS versions.

Beigesoft™ EIS collects (stores) your data only inside database file. Log files might contain private data.

If you set option Show debug messages to ON, then log files logStd0.log and logStd1.log will contain private data. In that case, if you are going to send these log files to a programmer (who customize this application), then you hove to remove private data from these files.

This is WEB-application, so you should use the newest version of WEB-browser. You should avoid to install any WEB-browser plugins.

Beigesoft™ EIS standalone versions.

These versions are based on embedded private A-Jetty WEB-server. This private WEB-server is not available from outside your computer (or Android device). These versions require authorization with strong password (created by you). They allow maximum 6 fail login attempts. The session will be blocked in 15 minutes after 6th fail login attempts. You should periodically check file logSec.log that contains authorization information. These versions use modern encryption algorithms with strong keys for HTTPS connection and for safe encrypted file exchanging. These are multi-database versions, you can create new and switch the database from WEB-interface. If you a going to sent database file to another computer then you should use WEB-interface for encrypting this file.

Beigesoft™ EIS on embedded A-Jetty for ordinal computer.

Your data is in database file, default is webapp/bseis.sqlite. You are able to copy (backup) database to a private place by using ordinal file manager. Keystore with secret keys for encryption is in ks folder. It is protected with strong password (created by you). You should set strong password for your OS account. It's advisable to set startup BIOS password on your computer.

Beigesoft™ EIS on embedded A-Jetty for Android.

Your data (SQLite databases), logStd.log, logSec.log, keystore (with secret keys) are in the private application storage. There is no way to read these files from there. Only this application can read and write these files. You can lost your data if you forget password (for keystore or/and application), so you should backup database files by using WEB-interface. This requires Beige-EIS standalone version on another computer (or Android device), cause application encrypts files with public key that belong to another Beige-EIS. Only another Beige-EIS (whose public key used for encryption) can decrypts these encrypted files. PDF reports will be downloaded into shared storage (external, USB, CD-card), so avoid to install any unnecessary application that requires permission to this storage. In the next year all Android applications can access any file on shared storage only when user allows it interactively, that is user will be asked about opening a file, so there will be no way of hidden reads of your files on shared storage. But if you connect your Android device to a computer in file transfer mode, then any application on that computer can read your files directly (in hidden way).

Beigesoft™ EIS cloud version.

Cloud (Internet available) version is more difficult to keep your data private, so use it only when it's necessary. You must yourself handle strong passwords and reliable HTTPS connection.
